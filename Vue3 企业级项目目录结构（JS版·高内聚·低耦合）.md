# Vue3 企业级项目目录结构（JS版·高内聚·低耦合）

适配技术栈：Vue3 \+ JavaScript \+ Pinia \+ Vue Router 4 \+ Vite \+ Axios，支持中大型项目、多团队协作、长期迭代维护，严格遵循「高内聚、低耦合」核心原则，兼顾工程化、可扩展性、可维护性，移除TS相关依赖，适配纯JS开发场景。

核心设计：按「业务模块」聚合相关代码，按「职责分层」解耦依赖，底层工具不依赖上层业务，上层业务可独立拆分、部署，私有资源不污染全局，公共资源统一管理，适配JS项目的开发习惯和维护需求。

## 企业级核心目录说明（高内聚·低耦合关键）

### 1\. 核心原则落地：按「业务模块」聚合，按「职责分层」解耦

✅ 高内聚：每个业务模块（如user、order）的页面、私有组件、业务逻辑，都聚合在对应目录下，删除/修改模块只需操作一个文件夹，不影响其他模块。

✅ 低耦合：模块之间不直接引用，通过api（接口）、store（状态）、utils（工具）通信；底层工具（utils、hooks）不依赖上层业务，可独立复用，适配JS项目的开发节奏。

### 2\. 关键目录职责（企业级必备·JS版适配）

- **api/**：完全解耦视图与接口，所有请求统一管理，JS版本移除TS类型定义，简化拦截器配置，修改接口地址、请求参数，无需改动页面代码；拦截器统一处理token、错误，减少重复代码。

- **composables/**：Vue3核心复用方案，JS版本保留组合式函数的逻辑复用优势，将页面逻辑抽离，页面只做渲染，逻辑可复用、可维护，避免页面组件过于臃肿（企业级项目核心优化点）。

- **pages/**：业务核心，按模块拆分，每个模块内部包含私有组件，私有组件不对外暴露，避免全局组件泛滥，提升维护性，JS项目可直接在此目录编写页面逻辑，无需额外类型定义。

- **stores/**：按模块拆分状态，不写巨型store，每个模块只管理自己的状态，避免状态混乱；通过Pinia的getters、actions通信，解耦模块状态，JS版本简化状态定义，无需TS类型约束。

- **constants/**：常量、枚举统一管理（JS用对象模拟枚举），禁止在页面中写魔法字符串（如“1”代表已支付），便于后期修改，适配JS项目的常量管理习惯。

- **permission/**：独立权限层，集中管理权限逻辑，页面、路由、组件的权限判断，都统一调用此处方法，避免权限逻辑散落在各个页面，JS版本简化逻辑，无需类型校验。

### 3\. 企业级额外规范（保障低耦合·JS版适配）

1. 私有组件必须放在对应pages/xxx/components下，严禁放入全局components，避免组件污染，JS项目无需额外配置类型，直接复用组件即可。

2. 页面不直接写Axios请求，必须通过api/modules下的接口方法，便于统一维护和Mock数据，JS项目可简化请求参数处理，无需类型定义。

3. 常量、枚举统一放在constants下，禁止在页面中写魔法字符串，便于后期修改，JS项目用对象模拟枚举，简化配置。

4. 路由、Pinia、指令等全局资源，统一在对应目录的index\.js导出注册，避免在main\.js中写大量注册代码，适配JS项目的模块化导出习惯。

5. 工具函数必须是纯函数（无副作用、输入输出一致），不依赖页面、store，可独立测试和复用，JS项目无需类型约束，重点保证函数复用性和无副作用。
