# 商家端目录架构说明

本文档对应项目：`vue/foodie_merchant`

当前商家端已经按 Vue3 模块化结构完成整理，核心分层为：

- `config`：应用级配置
- `api/request`：请求实例与拦截器
- `api/modules`：业务 API 统一出口
- `stores/modules`：Pinia 状态模块
- `router/modules`：路由模块拆分
- `layouts`：布局层
- `pages`：页面层
- `composables/business`：页面业务逻辑层
- `components`：可复用组件
- `services`：外部服务封装

## 根目录文件

- `.env.development`：开发环境变量，通常放接口地址、高德地图 Key 等开发态配置。
- `.gitignore`：Git 忽略规则。
- `README.md`：项目说明文件。
- `index.html`：Vite 应用 HTML 模板入口。
- `package-lock.json`：npm 依赖锁定文件，保证安装版本稳定。
- `package.json`：项目依赖、脚本命令与构建配置入口。
- `vite.config.js`：Vite 构建配置文件。
- `商家端目录架构说明.md`：当前这份商家端目录结构说明文档。

## src 目录总览

- `src/App.vue`：应用根组件，承接路由视图和全局壳层。
- `src/main.js`：应用启动入口，负责挂载 Vue、Pinia、Router 和全局样式。
- `src/style.css`：全局公共样式文件。

## src/api

- `src/api/auth.js`：认证相关原始 API 实现，供 `modules/auth.js` 兼容导出。
- `src/api/category.js`：商品分类/套餐分类相关原始 API 实现。
- `src/api/dish.js`：菜品管理相关原始 API 实现。
- `src/api/employee.js`：商户员工管理相关原始 API 实现。
- `src/api/finance.js`：财务结算、收入明细相关原始 API 实现。
- `src/api/merchant.js`：商户资料、营业状态、营业时间等相关原始 API 实现。
- `src/api/order.js`：订单分页、详情、接单、拒单等相关原始 API 实现。
- `src/api/review.js`：顾客评价与商家回复相关原始 API 实现。
- `src/api/setmeal.js`：套餐列表、详情、状态变更相关原始 API 实现。
- `src/api/statistics.js`：统计报表相关原始 API 实现。

### src/api/modules

- `src/api/modules/auth.js`：认证 API 模块统一出口，当前是对 `../auth` 的再导出。
- `src/api/modules/category.js`：分类 API 模块统一出口。
- `src/api/modules/dish.js`：菜品 API 模块统一出口。
- `src/api/modules/employee.js`：员工 API 模块统一出口。
- `src/api/modules/finance.js`：财务 API 模块统一出口。
- `src/api/modules/merchant.js`：商户 API 模块统一出口。
- `src/api/modules/order.js`：订单 API 模块统一出口。
- `src/api/modules/review.js`：评价 API 模块统一出口。
- `src/api/modules/setmeal.js`：套餐 API 模块统一出口。
- `src/api/modules/statistics.js`：统计 API 模块统一出口。

### src/api/request

- `src/api/request/index.js`：Axios 请求实例主入口，统一超时、基础配置和导出。
- `src/api/request/interceptors.js`：请求/响应拦截器，处理 token 注入、错误提示、登录失效跳转等。

## src/assets

- `src/assets/logo.svg`：项目默认 Logo 资源。

## src/components

- `src/components/DishForm.vue`：菜品新增/编辑弹窗组件，包含图片上传、分类选择、口味配置等。
- `src/components/DishSelector.vue`：套餐中选择菜品的双栏选择组件。
- `src/components/FlavorConfig.vue`：菜品口味配置组件，负责口味项的新增、编辑与删除。
- `src/components/UnderConstructionState.vue`：功能建设中占位组件，用于暂未完成的页面统一展示。

## src/composables

- `src/composables/useAuth.js`：认证相关通用 composable，负责登录、注册、退出等基础逻辑。
- `src/composables/useCategory.js`：分类管理通用 composable，负责分类列表、弹窗表单、状态切换等。
- `src/composables/useDashboardCharts.js`：首页图表相关通用逻辑封装。
- `src/composables/useProduct.js`：商品管理通用 composable，负责商品列表、状态切换、删除等逻辑。
- `src/composables/useTimer.js`：计时/定时相关通用逻辑封装。

### src/composables/business

- `src/composables/business/useCategoryPage.js`：分类管理页面级逻辑入口，对 `useCategory` 做页面层封装。
- `src/composables/business/useCustomerReviewPage.js`：顾客评价页业务逻辑，负责筛选、分页、回复弹窗等。
- `src/composables/business/useDashboardPage.js`：首页业务逻辑，负责统计数据、图表等页面状态。
- `src/composables/business/useFinanceIncomePage.js`：收入明细页业务逻辑，负责筛选、时间范围和分页请求。
- `src/composables/business/useFinanceSettlementPage.js`：结算记录页业务逻辑，负责结算周期筛选和分页请求。
- `src/composables/business/useLoginPage.js`：登录页业务逻辑，负责表单、校验、登录提交与状态同步。
- `src/composables/business/useMerchantProfileMap.js`：商户资料页地图逻辑，负责地图初始化、选点、逆地理解析等。
- `src/composables/business/useMerchantProfilePage.js`：商户资料页业务逻辑，负责资料加载、保存、营业状态与时间维护。
- `src/composables/business/useMerchantRegisterMap.js`：注册页地图逻辑，负责门店地址定位与地图交互。
- `src/composables/business/useOrderListPage.js`：订单列表页业务逻辑，负责筛选、详情、订单状态操作与统计数字。
- `src/composables/business/useProductListPage.js`：商品列表页业务逻辑入口，对 `useProduct` 做页面层封装。
- `src/composables/business/useRegisterPage.js`：注册页业务逻辑，负责注册表单与提交。
- `src/composables/business/useSetmealForm.js`：套餐表单业务逻辑，负责详情加载、图片上传、分类与菜品数据处理。
- `src/composables/business/useSetmealListPage.js`：套餐列表页业务逻辑，负责筛选、分页、批量删除、状态切换。
- `src/composables/business/useStaffPage.js`：员工管理页业务逻辑，负责列表、弹窗表单、重置密码、启停用等。

## src/config

- `src/config/env.js`：环境变量统一读取入口，如接口基础地址和高德地图安全配置。
- `src/config/pinia.js`：Pinia 配置导出入口。
- `src/config/router.js`：路由基础配置，如登录重定向等全局设置。

## src/constants

## src/layouts

### src/layouts/components

## src/stores

- `src/stores/index.js`：Pinia 实例创建文件，属于基础状态管理入口。

### src/stores/modules

- `src/stores/modules/dashboardStore.js`：首页统计数据状态模块。
- `src/stores/modules/merchantStore.js`：商户登录态、商户信息、token 等核心状态模块。

## src/utils

- `src/utils/request.js`：请求工具兼容出口，当前统一转发到 `@/api/request`。

## 现阶段结构说明

- 当前 `src/pages`、`src/layouts`、`src/router/modules`、`src/composables/business` 已经是商家端主结构。
- `src/api/*.js` 仍然保留原始实现，`src/api/modules/*.js` 负责作为新结构统一出口，这样可以兼容旧调用并逐步收口。
- `src/composables/useAuth.js`、`src/composables/useCategory.js`、`src/composables/useProduct.js` 属于可复用基础逻辑层，页面层优先通过 `src/composables/business` 访问。
- 部分页面如库存、审核、售后、报表等当前仍是占位页，但已经进入新目录结构，后续可直接在对应页面下继续实现。
