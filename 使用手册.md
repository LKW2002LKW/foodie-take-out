# 吃货联盟项目用户手册

## 一、手册说明

本手册面向普通使用者，适用于本项目在本地电脑或服务器上的普通运行方式，不涉及 Docker 部署。

本项目包含三套前端和三套后端：

- 平台端前端：`vue/foodie_plateform`
- 商家端前端：`vue/foodie_merchant`
- 用户端前端：`vue/my_foodie_app`
- 平台端后端：`foodie-platform-server`
- 商家端后端：`foodie-merchant-server`
- 用户端后端：`foodie-user-server`

## 二、运行环境要求

建议准备以下软件：

- JDK 17
- Maven 3.8 及以上
- Node.js 18 或 20
- npm 9 及以上
- MySQL 8.x
- Redis 6.x 或 7.x
- MinIO

## 三、项目访问端口

### 1. 后端端口

- 平台端后端：`8081`
- 商家端后端：`8082`
- 用户端后端：`8083`

### 2. 前端端口

- 平台端前端：`5343`
- 商家端前端：`5342`
- 用户端前端：`5341`

### 3. 基础服务端口

- MySQL：`3306`
- Redis：`6379`
- MinIO：`9000`
- MinIO 控制台：`9001`

## 四、启动前准备

### 1. 数据库准备

项目默认数据库名为：

```text
foodie_alliance
```

你现在已经把完整数据库脚本放在：

- [sql/foodie_alliance.sql](/mnt/d/baidunetdiskdownload/黑马/苍穹/项目/foodie-take-out/sql/foodie_alliance.sql)

建议先在 MySQL 中创建数据库：

```sql
CREATE DATABASE foodie_alliance DEFAULT CHARACTER SET utf8mb4;
```

然后导入脚本：

```bash
mysql -h 127.0.0.1 -P 3306 -uroot -p foodie_alliance < sql/foodie_alliance.sql
```

### 2. Redis 准备

启动 Redis 服务，默认端口为 `6379`。

### 3. MinIO 准备

启动 MinIO，并创建桶：

```text
foodie-images
```

项目图片默认上传到该桶。

### 2. 使用环境变量覆盖

也可以用环境变量覆盖，例如：

```bash
export DB_HOST=127.0.0.1
export DB_PORT=3306
export DB_NAME=foodie_alliance
export DB_USERNAME=root
export DB_PASSWORD=你的数据库密码

export REDIS_HOST=127.0.0.1
export REDIS_PORT=6379
export REDIS_PASSWORD=你的Redis密码

export MINIO_ENDPOINT=http://127.0.0.1:9000
export MINIO_ACCESS_KEY=admin
export MINIO_SECRET_KEY=你的MinIO密码
export MINIO_BUCKET=foodie-images
export MINIO_PUBLIC_URL=http://127.0.0.1:9000/foodie-images
```

## 六、后端启动方式

在项目根目录执行。

### 1. 编译项目

```bash
mvn clean install -DskipTests
```

### 2. 启动三个后端

启动平台端：

```bash
mvn -pl foodie-platform-server spring-boot:run
```

启动商家端：

```bash
mvn -pl foodie-merchant-server spring-boot:run
```

启动用户端：

```bash
mvn -pl foodie-user-server spring-boot:run
```

### 3. 启动成功后访问

- 平台端接口文档：`http://localhost:8081/doc.html`
- 商家端接口文档：`http://localhost:8082/doc.html`
- 用户端接口文档：`http://localhost:8083/doc.html`

## 十二、常见问题排查

### 1. 前端打开后接口报错

优先检查：

- 对应后端是否已经启动
- 后端端口是否被占用
- 前端访问地址是否正确

### 2. 后端启动时报数据库连接失败

优先检查：

- MySQL 是否启动
- 数据库 `foodie_alliance` 是否存在
- SQL 是否已经导入
- 用户名密码是否正确
- 配置文件或环境变量是否正确

### 3. Redis 连接失败

优先检查：

- Redis 是否启动
- 端口是否正确
- 密码是否正确

### 4. 图片上传失败

优先检查：

- MinIO 是否启动
- `foodie-images` 桶是否存在
- `MINIO_ENDPOINT` 是否正确
- `MINIO_PUBLIC_URL` 是否正确

### 5. 用户端地图不可用

优先检查：

- 是否配置了高德地图 Key
- 前端环境变量是否正确
